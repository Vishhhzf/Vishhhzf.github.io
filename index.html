SISTEMAS

01

FoxHelp Web Hub

Cómo se trabajó: Diseñado bajo metodologías ágiles de ingeniería de software aplicando los principios de Clean Architecture y Feature Sliced Design (FSD). La capa de infraestructura de datos se desacopló por completo de la capa de renderizado mediante adaptadores estructurados (Gateways), permitiendo una escalabilidad impecable del sistema.

Qué posee: Sincronización asíncrona reactiva del estado del frente de guerra, radar dinámico satelital vectorizado sobre un lienzo <canvas> HTML5 optimizado con bucles de renderizado a 60 FPS fijos, colas globales de suministro logístico y telemetría de red.

El resultado: Una plataforma web robusta capaz de actualizar y procesar flujos de telemetría militar en tiempo real en menos de 100ms, integrando CSP rígidas y defensas Google App Check para prevenir brechas de seguridad.

ES Modules Firebase SDK App Check
02

FoxOverlays HUD Client

Cómo se trabajó: Desarrollado sobre el framework Electron API, implementando puentes de comunicación IPC (`ipcRenderer` e `ipcMain`) ultra seguros mediante el uso estricto de secuencias de comandos de precarga (Preload) aisladas de contexto. Para el renderizado visual, se implementó aceleración por GPU mediante CSS 3D Transforms para no afectar el rendimiento nativo de los videojuegos de fondo.

Qué posee: Capas de visualización transparentes con atajos de teclado globales registrados dinámicamente, soporte nativo de interacción de clic a través de la ventana (Click-Through behavior), renderizadores vectoriales de radar coordinados y buffers de recarga.

El resultado: Un HUD de juego inmersivo de latencia cero que consume menos del 1.5% de CPU local, permitiendo a los operativos monitorear telemetría de red de manera integrada sin caídas de FPS.

Electron API GPU Compositing Global Shortcuts
03

FoxAndroid Field Companion

Cómo se trabajó: Programado nativamente en **Kotlin y Java** utilizando la arquitectura reactiva recomendada por Google: MVVM (Model-View-ViewModel) con LiveData. Los flujos de datos asíncronos se manejan a través de listeners continuos que consumen la base de datos en la nube de Firebase, estructurando una base robusta y limpia.

Qué posee: Sincronización reactiva del inventario logístico del frente, un motor local de notificaciones push de alta prioridad para llamadas de emergencia tácticas, calculadoras de recursos optimizadas y un sistema de caché de persistencia local offline-first.

El resultado: Una aplicación móvil ágil que permite a los operativos en el campo de batalla reaccionar a incidentes, coordinar rutas de convoyes logísticos y consultar manuales de entrenamiento sin consumir plan de datos excesivo.

Kotlin / Java Push Alerts Offline Cache
04

FoxExtension Browser Companion

Cómo se trabajó: Desarrollado bajo las directivas del estándar moderno de extensiones de Google Chrome: Manifest V3. Se implementaron flujos asíncronos mediante trabajadores en segundo plano (Service Workers) y scripts de inyección de contenido (Content Scripts) optimizados para no generar bloqueos del hilo principal del navegador.

Qué posee: Ventana emergente (Popup) con calculadoras de conversión logística avanzadas, herramientas de análisis automático de datos inyectados en páginas externas e integración directa con las colas globales del Hub Central de FoxHelp.

El resultado: Una extensión de navegador segura y sumamente veloz que ahorra a los planificadores de red tener que cambiar de pestaña o app externa, automatizando el flujo de trabajo operacional en un solo clic.
